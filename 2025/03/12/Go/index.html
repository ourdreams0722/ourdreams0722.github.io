编译与运行

  • Q：WINDOWS下隐藏exe可执行文件弹出的输出黑窗口

编译时增加编译参数：go build -ldflags “-H windowsgui”

基础语法

GO中不需要有; 编译器会自动处理，如果想要一行内写多个语句可以使用;分隔

‘{‘不能单独一行，否则报错

return可以return多个值，在调用接收时必须同样使用多个变量接收

接收变量可以使用’_’作为变量名来接收不需要使用的值

变量声明

​ 批量声明时如果为赋值，其会默认与上一行值一致

变量

​ var 变量名 变量类型

​ Go推荐使用驼峰命名，非驼峰会标记不规范命名

​ 被声明的变量必须被引用，否则报错

​ 批量声明：var (

​ 变量a 变量类型

​ 变量b 变量类型

​ )

​ 简短声明：变量a,变量b := 值a,值b

​ 简短声明使用:= 批量时key的数量必须与value数量对应否 则报错,value如果比key少则视为key的value都为此值，只能 在函数中使用简短声明

​ %s 占位符，在后面声明需要填充的值

​ (“name:%s”,小蓝) 输入：name:小蓝

​ 匿名变量：’_’一个单独的下划线，表示忽略某个值

常量

​ const 声明常量，通常定义为全局

​ 常量定义之后不能被修改

​ iota 类似于枚举，常数每声明一个iota计数+1，iota在const中 出现时其会被重置为0

切片

​ 切片初始化

初始化切片s,是数组arr的引用

len() 和 cap() 函数

切片是可索引的，并且可以由 len() 方法获取长度。

切片提供了计算容量的方法 cap() 可以测量切片最长可以达到多少。

截取切片与截取数组赋给切片

切片名[起始索引:结束索引]

Map

​ 两种创建方式：var|make

​ 删除集合中的值：

​ delete(集合名,key)

类型转换

​ 转换后类型(转换值)

​ var a int = 3

​ 例：float32(a)

循环&&判断

​ 跳出多层for循环：

​ ①. 外部定义变量，内部if修改变量，外层循环if判断变量

​ ②. 底部定义一个标签 例如 XX: 循环中使用 goto 标签名(XX) 来跳出循环去往标签位置

并发特性

​ Go出现时已有双核处理器出现，所以它天生支持并发，不需要去额外的引入类库

使用保留字 go

​ go 函数名(参数)

函数特性

​ 匿名函数：

​ 未命名函数，分两种定义方式：

​ ①.将匿名函数赋给变量 a=func(){return 1}

​ ②.结尾加一个括号表示立即执行：func(){return 1}()

​ 闭包：

​ 闭包必须是匿名函数，判断其是否为闭包依据其内部是否引用了函数外的变量

​ func 函数a (){

​ var 变量a 变量类型

​ func (){

​ return 变量a

​ }

​ }

异常处理

​ go中没有try catch它使用recover，panic来处理异常

​ panic定义异常，recover捕获异常

​ defer声明在函数即将结束时执行

​ 例：defer func(){

​ 错误a := recover()

​ if 错误a != nil{

​ //抛出信息

​ }

​ panic(“panic用来定义产生异常”)

​ }

注：recover()必须搭配defer使用
defer一定要在可能引发pinic的语句前定义

结构体与实例化

类似于java，php中的模型实体类概念

​ type 类型名 struct{

​ 变量名 变量类型

​ }

变量类型相同的变量尽量放在一起批量定义，有助于内存优化 例：变量a,变量b string

实例化：结构体定义后并不会被分配内存，需要实例化分配内存

​ var 变量名 结构体类型名

​ 结构体内部变量赋值：变量名.结构体内变量名=值

结构体实例化后再内存中占用的是一块连续的内存地址

仅使用一次的结构体定义

​ 不使用type关键字使用var：var 类型名 struct{…}

指针结构体定义

​ 创建不变，实例化时new出来：var 变量名=new(结构体类型名)

​ 指针结构体在赋值时要加*,例：*结构体名.内部变量名=值;但go编译时会自动对指针结构体加上*所以直接 “结构体名.变量名” 即可

键值对形式的初始化

​ 变量名 := 结构体名{

​ 变量a : 值,

​ 变量b : 值,

​ }

列表形式初始化

​ 按照结构体内顺序挨个赋值

​ 变量名 := 结构体名{

​ 值a,

​ 值b,

​ }

构造函数

​ go中没有构造函数，可以使用结构体去自定义实现构造函数

​ 例：func newTest结构体(参数列表也是结构体的内部列表) 返回值类型结构体名 {

​ return 返回值类型结构体名 {

​ 参数列表也是结构体的内部列表

​ }

​ }

​ 这样的构造函数会直接实例化所占内存开销较大，使用指针结构体,*,&定义

​ 例：func newTest结构体(参数列表也是结构体的内部列表) *返回值类型结构体名 {

​ return &返回值类型结构体名 {

​ 参数列表也是结构体的内部列表

​ }

​ }

方法

需要接收者与接收者类型，一般接收者命名为接收者类型首字母小写

例：func (p Person) 方法名(){

​ //业务

}

其中Person为已定义好的结构体

在需要修改结构体时必须使用指针接收者，值接收者只能copy不能修改

接收者类型可以为指针接收者(加*号)，值接收者

为任意类型定义接收者

​ 如果接收者类型为基础类型的话，需要基于内置基本类型定义一个自定义类型,接收者类型使用此类型

Go中的public

​ 变量名与结构体中变量名在命名时如果 首字母小写表示仅本包可见，首字母大写为公开的，外部可见，函数方法也相同

JSON

​ 序列化：data, err := json.Marshal(需要序列化值)

​ 反序列化：err = json.Unmarshal([]byte(需要反序列化值), 成功后赋值的变量)

​ 在序列化与反序列化时调用结构体的内部属性时仅可调用首字母大写变量，因为首字母小写为私有变量

​ 解决：使用Tag定义别名

​ Tag是结构体的元信息，可以在运行的时候通过反射的机制读取出来。 Tag在结构体字段的后方定义，由一对反引号包裹起来，具体的格式如下：

结构体标签由一个或多个键值对组成。键与值使用冒号分隔，值用双引号括起来。键值对之间使用一个空格分隔。 注意事项： 为结构体编写Tag时，必须严格遵守键值对的规则。结构体标签的解析代码的容错能力很差，一旦格式写错，编译和运行时都不会提示任何错误，通过反射也无法正确取值。例如不要在key和value之间添加空格。

上述定义了tag为json，db(数据库)，xml格式的别名 定义多个使用空格间隔
